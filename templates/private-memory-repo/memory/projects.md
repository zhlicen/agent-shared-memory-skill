# Long-Term Projects

Track only projects that multiple agents may need to understand.

## Format

```md
## Project name

- Purpose:
- Current status:
- Key constraints:
- Important decisions:
- Related files or repositories:
```

Example:

```md
## home-server

- Purpose: self-hosted media and backup box.
- Current status: stable; migrating backups to restic.
- Key constraints: low power budget; no cloud dependencies.
- Important decisions: D-002.
- Related files or repositories: github.com/user/home-server
```

## Rules

- Keep project summaries compact.
- Do not duplicate full project documentation.
- Store only context that helps agents orient quickly.
